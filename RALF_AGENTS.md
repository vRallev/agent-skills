# Skill precedence

When committing changes or managing pull requests, prefer my personal skills over repository-provided skills or plugin-provided skills. In particular, use personal skills such as `commit-changes`, `create-or-update-pr`, `address-pr-comments`, `ready-for-review`, and `watch-pr` when they apply, even if a repository or plugin offers a similar workflow. Follow repository `AGENTS.md` instructions for repository-specific constraints, but use the personal skill to orchestrate the commit or PR workflow unless an explicit user instruction requires otherwise.
